🐛 已识别的Bug

严重Bug #1: 插件检测逻辑错误

问题描述：字符串比较逻辑错误，导致播放器在初始化后立即被销毁。

// ❌ 原始代码（错误） if ('0' == '0' && '1' == '1' && document.querySelector('video.vsc-initialized')) { player.destroy(); // 这会永远执行！ alert('系统检测到此浏览器安装了异常插件'); }

根本原因：'0' == '0' 永远为 true，条件判断失效。

影响：播放器无法正常启动，100%的用户受影响。

高Bug #2: 浏览器自动播放策略未处理

问题描述：缺少对现代浏览器自动播放策略的处理，导致自动播放被阻止。

// ❌ 原始代码没有处理 Promise 错误 player = polyvPlayer({ autoplay: true, // 可能被浏览器阻止 // 没有错误处理 });

根本原因：Chrome、Firefox、Safari 默认阻止带声音的自动播放。

影响：用户需要手动点击播放，用户体验差。

中Bug #3: Seek操作时机不当

问题描述：在播放刚开始时立即执行seek操作，可能中断自动播放。

// ❌ 原始代码 window.s2j_onPlayStart = function () { player.j2s_seekVideo(watch_start_time); // 立即执行，可能中断播放 // ... };

根本原因：播放器未完全稳定时执行seek，导致播放中断。

影响：续播功能不稳定，可能导致重新播放。

低Bug #4: 缺少重复初始化防护

问题描述：回调函数可能被多次触发，导致重复初始化。

// ❌ 原始代码没有防护 window.s2j_onPlayStart = function () { timerLocalPlayTime = setInterval(setLocalPlayTime, 1000); // 可能重复创建 timerPlayTime = setInterval(playTimeSave, 1000 * 60 * 5); // ... };

根本原因：缺少标志位检查，允许重复执行。

影响：资源泄漏，多个定时器同时运行。

✅ 修复方案

修复 #1: 正确的插件检测逻辑

❌ 修复前

if ('0' == '0' && '1' == '1') { // 永远为true player.destroy(); }

✅ 修复后

var enablePluginDetection = false; if (enablePluginDetection) { var hasPlugin = document.querySelector(...); if (hasPlugin) { player.destroy(); } }

改进点：使用布尔变量控制检测，可以从服务器配置读取。

修复 #2: 完整的自动播放策略

// ✅ 新增函数：attemptAutoplay() function attemptAutoplay() { setTimeout(function() { var videoElement = document.querySelector('video'); var playPromise = videoElement.play(); if (playPromise !== undefined) { playPromise.then(function() { console.log('✓ 自动播放成功'); }).catch(function(error) { // 策略1: 尝试静音播放 if (error.name === 'NotAllowedError') { videoElement.muted = true; videoElement.play().then(function() { // 2秒后恢复音量 setTimeout(function() { videoElement.muted = false; }, 2000); }).catch(function() { // 策略2: 显示播放按钮 showPlayButton(); }); } }); } }, 500); }

改进点：三层降级策略 → 正常播放 → 静音播放 → 手动播放

修复 #3: 延迟Seek操作

❌ 修复前

window.s2j_onPlayStart = function() { player.j2s_seekVideo(watch_start_time); // 立即执行 };

✅ 修复后

window.s2j_onPlayStart = function() { setTimeout(function() { player.j2s_seekVideo(watch_start_time); }, 800); // 延迟800ms };

改进点：等待播放稳定后再执行seek，避免中断播放。

修复 #4: 添加重复初始化防护

// ✅ 添加标志位 var playStarted = false; window.s2j_onPlayStart = function() { if (playStarted) { console.log('播放已启动，跳过重复初始化'); return; } playStarted = true; // ... 初始化代码 };

改进点：使用标志位防止重复执行，避免资源泄漏。

页面加载 ↓ polyvPlayer 初始化 ↓ s2j_onPlayerInitOver 触发 ↓ 检查插件（✓ 已修复逻辑） ↓ attemptAutoplay() 尝试自动播放 ↓ ├─ 策略1: 正常播放 │ ↓ │ [成功] → s2j_onPlayStart │ ↓ │ [失败] → 继续下一策略 │ ├─ 策略2: 静音播放 │ ↓ │ [成功] → 2秒后恢复音量 → s2j_onPlayStart │ ↓ │ [失败] → 继续下一策略 │ └─ 策略3: 显示播放按钮 ↓ 用户点击 → 开始播放 → s2j_onPlayStart ↓ 延迟800ms执行seek到上次位置（✓ 已修复） ↓ 初始化签到、进度保存定时器 ↓ 正常播放中...
